## 1. 几个基础概念

在开始之前，用最直白的方式告诉你这些名词是什么。

### 这个项目到底是个什么东西？

简单说：它是一个**把你的读书笔记变成漂亮网站**的工具。

你用最普通的文本写笔记，它帮你把笔记排版、生成知识图谱、加上全文搜索——最后变成一个可以在浏览器里打开的网站。

### Node.js 是什么？

**Node.js** 是一个让计算机能运行 JavaScript 的程序。你可以把它理解成一个"播放器"：

- `.mp4` 文件需要播放器（比如 VLC）才能播放
- 这个项目需要 Node.js 才能运行

> **你不需要学会写 JavaScript，就像你不需要会拍电影才能打开播放器看电影。**

你只需要装好 Node.js，然后执行几条固定的命令，项目就自动跑起来了。

### npm 是什么？

**npm** 是 Node.js 自带的「应用商店」。里面全世界的程序员上传了各种工具包。

- `npm install` = 到这个项目的文件夹里，一键下载所有需要的工具
- `npm run dev` = 运行项目，启动预览

> **类比：** npm 就像手机应用商店，`npm install` 是搜索安装 App，`npm run dev` 是点开 App 使用。

### npx 是什么？

**npx** 是一个"临时下载并运行"的工具。用完就自动清理，不占空间。

> **类比：** `npm install` 是买一本书永久收藏，`npx` 是从图书馆借一本书，看完就还。

### GitHub / Git 是什么？

**Git** 是一个「版本管理工具」，用来记录你对文件做过的每一次修改。
**GitHub** 是一个「代码存管网站」，把你的文件存到网上，方便分享和备份。

这个项目的笔记想发布到网上让人访问，就需要用到 GitHub。

## 2. 环境准备：安装 Node.js

> ✅ **先检查有没有装过：**
> 按 `Win + R`，输入 `cmd` 回车，在黑色窗口中输入 `node -v` 回车。
> - 如果显示 `v22.x.x` 或更高 → ✅ 已经装好了，跳过这一步
> - 如果提示"不是内部或外部命令" → 继续往下看

### 下载安装

1. 打开 https://nodejs.org
2. 你会看到两个下载按钮，选左侧的 **LTS（长期支持版）**，点击下载
3. 下载完成后，双击安装包
4. 一路点 **Next**（下一步），直到安装完成（所有选项保持默认即可）
5. 安装完成后，重启一下 CMD 窗口，输入 `node -v` 验证是否装好

> ✅ 装好 Node.js 之后，npm 会自动一起装好，不需要单独安装。

## 3. 把项目跑起来

### 打开 CMD（命令提示符）

> **⚠️ 重要：** 本项目要用 **CMD（命令提示符）** 来操作，不要用 PowerShell。

打开方式（任选一种）：
- 按 `Win + R`，输入 `cmd`，回车
- 或在 VSCode 中按 `` Ctrl + ` ``（键盘左上角 Esc 下方那个键），确保终端右上角显示 `cmd` 或 `Command Prompt`

### 进入项目目录

在 CMD 中输入以下命令，回车：

```cmd
cd d:\interest\writting
```

> 这行命令的意思是"进入项目所在的文件夹"。之后所有操作都在这个目录下进行。

### 第一步：安装依赖

```cmd
npm install
```

这条命令会从网上下载项目所需的全部工具。根据网速快慢，可能需要 30 秒到 2 分钟。

- 下载过程中会显示一些滚动文字，不用管
- 等到光标重新闪烁，没有报错红色文字，就说明安装完成了
- **这一步只需运行一次**，以后只要不删除项目，不用再运行

### 第二步：启动本地预览

```cmd
npm run dev
```

运行后会显示类似以下信息：

```
> quartz build --serve

Cleaned output directory `public` in 13ms
Found 10 input files from `content` in 10ms
...
Done processing 10 files in 270ms
Serving on http://localhost:8080
```

现在打开浏览器，地址栏输入 **http://localhost:8080**，回车。

**你就能看到自己的读书笔记网站了！** 🎉

> 💡 **要点：**
> - 这个 CMD 窗口不能关，关了网站就停了
> - 想看的时候重新运行 `npm run dev` 即可

### 第三步：停止预览

在 CMD 窗口中按 `Ctrl + C`，然后按 `Y` 确认，即可停止。

## 4. 往项目里加笔记

### 笔记存放在哪里？

所有笔记都在 `content/` 文件夹下。每个笔记是一个 `.md` 文件（Markdown 文件）。

> **什么是 .md 文件？** 就是一种纯文本文件，用简单的符号标记格式。比如 `# 标题` 表示大标题，`**加粗**` 表示加粗。用 VSCode 打开就能直接编辑。

### 新建一条笔记

1. 在 `content/` 文件夹下新建一个文本文件
2. 文件名改成 `笔记标题.md`（例如 `秦汉爵位制度.md`）
3. 用 VSCode 打开，写入以下内容：

```markdown
---
title: 秦汉爵位制度
tags:
  - 制度
  - 秦汉
created: 2025-01-15
source: 《秦汉史》钱穆
---

# 秦汉爵位制度

## 核心内容

记录你读书的核心要点...

## 个人思考

你的理解和联想...

## 与小说创作的联系

这些知识怎么用在你的小说里...

## 相关笔记

- [[刘邦起于亭长]]
```

4. 保存文件
5. 如果你还开着 `npm run dev`，浏览器会自动刷新，新笔记立刻可见

### 关联笔记（双向链接）

在笔记中写 `[[另一篇笔记的标题]]`，两篇笔记就会自动产生关联：

```markdown
关于秦汉的选官制度，参见 [[察举制与征辟制]]
```

保存后，知识图谱中会自动显示这两篇笔记之间有连线，每篇笔记底部也会显示"被哪些笔记引用"。

### 打标签

通过 `tags` 字段给笔记打标签，方便跨主题检索：

```yaml
tags:
  - 秦汉
  - 官制
  - 爵位
```

系统会自动生成标签索引页，点击标签就能看到所有打了这个标签的笔记。

### 参考示例和模板

项目自带了一个示例笔记和模板文件：

- **示例：** `content/example-刘邦起于亭长.md` — 一篇完整的笔记，可以直接参考它的写法
- **模板：** `content/templates/笔记模板.md` — 空白的模板，新建笔记时可以直接复制

## 5. 发布到网上让别人也能看

如果你只想自己看，完成上面第 3 步就足够了。但你也可以把笔记发布到网上——手机、平板、朋友都能访问。

### 注册 GitHub

1. 打开 https://github.com 点右上角 **Sign up**
2. 输入邮箱、密码、用户名，按提示完成验证
3. 注册完成后登录

### 创建仓库

1. 登录后点击右上角 `+` → **New repository**
2. Repository name 输入 `reading-notes`（或者你喜欢的名字）
3. 可见性选 **Public**（公开）或 **Private**（私密，仅自己可见）
4. 其他保持默认，点 **Create repository**
5. 创建成功后你会看到一个页面，上面有几行命令。暂时放着，等会儿用。

### 把项目推送到 GitHub

在 CMD 中运行（确保已经在 `d:\interest\writting` 目录下）：

```cmd
git remote add origin https://github.com/你的用户名/reading-notes.git
git push -u origin master
```

> 🔑 **关于密码：** 第一次推送 GitHub 会要求验证身份。
> - 用户名输入你的 GitHub 用户名
> - 密码**不是**你的 GitHub 登录密码，而是一个叫 **Personal Access Token** 的东西
>
> **如何生成 Token：**
> 1. GitHub 右上角头像 → **Settings** → **Developer settings**
> 2. **Personal access tokens** → **Tokens (classic)** → **Generate new token**
> 3. 勾选 `repo` 权限，点 **Generate**
> 4. 复制生成的 token（一串字母数字，像这样：`ghp_xxxxxxxxxxxx`）
> 5. 在 CMD 提示输入密码时，右键粘贴这个 token 即可
>    （粘贴后屏幕不会显示任何内容，这是正常的，按回车就行）

### 开启自动部署

1. 在浏览器打开你的 GitHub 仓库页面
2. 点 **Settings**（齿轮图标）
3. 左侧菜单找到 **Pages**
4. Source 处选择 **GitHub Actions**
5. 项目已经自带了部署配置文件，以后每次推送，GitHub 会自动重新构建并发布

### 访问你的笔记网站

推送完成后，等 1-2 分钟，访问：

```
https://你的用户名.github.io/reading-notes
```

就能看到你的笔记网站了！🎉

## 6. 日常使用流程

### 纯本地使用（最简单，不发布到网上）

```
1. 打开 CMD
2. cd d:\interest\writting
3. npm run dev
4. 浏览器打开 http://localhost:8080
5. 在 VSCode 里编辑 content/ 下的笔记 → 浏览器自动刷新
6. 看完了按 Ctrl + C 停止
```

### 发布到网上（GitHub Pages）

```
1. 写笔记（在 VSCode 里编辑 content/ 下的 .md 文件）
2. 推送（把笔记同步到 GitHub）：
   git add .
   git commit -m "添加了XX笔记"
   git push
3. 等 1 分钟，网站自动更新
```

> 听起来复杂，但习惯后整个流程其实就是：**写 → 推**，两行命令搞定。

## 常见问题

### Q：CMD 报错"不是内部或外部命令"

说明 Node.js 没有安装或没有装对。检查一下：
- 是否下载了 LTS 版本？
- 安装时是否一路点 Next 没有取消勾选？
- 是否重启了 CMD 窗口？

### Q：PowerShell 报错"无法加载文件 npx.ps1"

解决方法：**不要用 PowerShell，改用 CMD（命令提示符）**。详见上面第 3 节。

### Q：npm install 报错

可能是网络问题。可以换国内镜像源加速：

```cmd
npm config set registry https://registry.npmmirror.com
```

设置后再运行 `npm install`。

### Q：npm run dev 后浏览器打开是白屏

- 检查地址是不是 http://localhost:8080
- 检查 CMD 窗口是否还在运行（没有关闭）
- 检查是否有其他程序占用了 8080 端口

### Q：我想重命名一篇笔记

直接改文件名就行。但需要手动更新其他笔记中引用它的 `[[旧名称]]` 为 `[[新名称]]`。

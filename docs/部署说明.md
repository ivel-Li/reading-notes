# 部署说明

三种部署方式的详细步骤。

---

## 方式一：纯本地使用

仅在自己电脑上查看，无需联网：

```bash
cd 你的项目路径
npm run dev
# → 浏览器打开 http://localhost:8080
# （例如：cd d:\interest\writting，替换成你的实际路径）
```

---

## 方式二：GitHub Pages

**前置条件：** 注册 [GitHub](https://github.com) 账号。

### 创建仓库

1. 登录 GitHub，点右上角 `+` → **New repository**
2. Repository name 输入 `reading-notes`，选 Public/Private
3. 点 **Create repository**

### 推送代码

```bash
git remote add origin https://github.com/你的用户名/reading-notes.git
git push -u origin master
```

> 首次推送身份验证需使用 **Personal Access Token**（非登录密码）：
> GitHub 右上角头像 → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → 勾选 `repo` → Generate → 复制 token 粘贴到密码提示框

### 开启 GitHub Actions 部署

1. 仓库页面 → **Settings** → **Pages**
2. Source 选择 **GitHub Actions**
3. 项目自带 `.github/workflows/deploy.yml`，GitHub 会自动识别并部署

### 访问

推送后约 1-2 分钟，访问：
```
https://你的用户名.github.io/reading-notes
```

### 后续日常更新

```bash
git add .
git commit -m "添加了XX笔记"
git push
```
推送后网站自动更新。

---

## 方式三：Docker 部署（NAS/自有服务器）

### 构建镜像

```bash
docker build -t reading-notes .
```

### 运行容器

```bash
docker run -d -p 8080:80 --name reading-notes reading-notes
```

然后访问 `http://你的服务器IP:8080`。

### 更新笔记后

```bash
npm run build        # 重新构建
docker build -t reading-notes .  # 重建镜像
docker stop reading-notes && docker rm reading-notes  # 停止旧容器
docker run -d -p 8080:80 --name reading-notes reading-notes  # 启动新容器
```

### 群晖 NAS 用户

也可使用 `deploy-nas.bat` 脚本（需先修改其中的 NAS 地址与路径）。
